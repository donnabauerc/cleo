# C.L.E.O.
### **C**onceptual **L**ow **E**arth **O**rbit **E**valuator

This repository serves as the **Proof of Concept (PoC)** for a Bachelor Thesis at the **Johannes Kepler University (JKU)**. It focuses on the technical integration and evaluation of satellite systems in low earth orbit.

---

## 📡 Project Overview
The primary goal of this project is to evaluate different approaches to handling **Low Earth Orbit (LEO)** satellite data. The practical implementation focuses on fetching, processing, and analyzing telemetry and geospatial data from various LEO constellations via REST APIs.

---
## 🛠 Tech Stack
* **Language:** Python 3.x
* **Data Sources (Planned):** Open-Notify (ISS), NASA Earth API, Copernicus Ecosystem.

---

## 🛰 The Acronym: Why C.L.E.O.?

The name **C.L.E.O.** was chosen to reflect both the scientific and the technical core of the thesis:

* **Conceptual:** Highlights the theoretical foundation and the comparative analysis of organizational frameworks.
* **Low Earth Orbit:** Defines the physical scope of the data. LEO satellites (like the ISS or Starlink) orbit at altitudes between 160 and 2,000 km, offering high-resolution data and low latency.
* **Evaluator:** Describes the main function of the software artifact—to test the feasibility and efficiency of the proposed organizational models.
